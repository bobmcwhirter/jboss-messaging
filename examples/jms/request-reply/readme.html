JMS Request-Reply Example


This example shows you how to send a request message and receive a reply. It first sends a request message with a JMSReplyTo header set to get a reply, then after receiving the request message, forges a reply message and send out to the reply queue. Then it receives the reply message.

Request/Reply style messaging is supported through standard JMS message headers JMSReplyTo and JMSCorrelationID. This is often used in request-reply style communications between applications. Whenever a client sends a message that expects a response, it can use this mechanism to implement. please consult the JMS 1.1 specification for full details.


Example step-by-step

To run the example, simply type ant from this directory


  1. First we need to get an initial context so we can look-up the JMS connection factory and destination objects from JNDI. This initial context will get it's properties from the client-jndi.properties file in the directory ../common/config
    2.            InitialContext initialContext = getContext();
  2. We lookup the queue for sending the request message
    3.            Queue requestQueue = (Queue) initialContext.lookup("/queue/exampleQueue");
  3. We lookup for the Connection Factory
    4.            ConnectionFactory cf = (ConnectionFactory) initialContext.lookup("/ConnectionFactory");
  4. We create a JMS Connection
    5.            connection = cf.createConnection();
  5. We start the connection
    6.            connection.start();
  6. We create a JMS Session
    7.            Session session = connection.createSession(false, Session.AUTO_ACKNOWLEDGE);
  7. We create a JMS Message Producer to send request message
    8.            MessageProducer producer = session.createProducer(requestQueue);
  8. We create a temporary queue used to send reply message to and receive reply from
    9.            TemporaryQueue replyQueue = session.createTemporaryQueue();
  9. We create a JMS Message Consumer to receive request
    10.            MessageConsumer messageConsumer = session.createConsumer(requestQueue);
  10. We create a request Text Message
    11.            TextMessage requestMsg = session.createTextMessage("A request message");
  11. We set the ReplyTo header so that the request receiver knows where to send the reply.
    12.            requestMsg.setJMSReplyTo(replyQueue);
  12. We set the CorrelationID so that it can be linked with corresponding reply message
    13.            requestMsg.setJMSCorrelationID("jbm-id: 0000001");
  13. We sent the request message
    14.            producer.send(requestMsg);
  14. We receive the request message
    15.            TextMessage requestMsgReceived = (TextMessage) messageConsumer.receive(5000);
  15. We extract the ReplyTo destination from the request message
    16.            Destination replyDestination = requestMsgReceived.getJMSReplyTo();
  16. We create a reply message
    17.            TextMessage replyMessage = session.createTextMessage("A reply message");
  17. We set the CorrelationID
    18.            replyMessage.setJMSCorrelationID(requestMsgReceived.getJMSCorrelationID());
  18. We create a producer to send the reply message
    19.            MessageProducer replyProducer = session.createProducer(replyDestination);
  19. We send out the reply message
    20.            replyProducer.send(replyMessage);
  20. We create consumer to receive reply message
    21.            MessageConsumer replyConsumer = session.createConsumer(replyDestination);
  21. We receive the reply message
    22.            TextMessage replyMessageReceived = (TextMessage)replyConsumer.receive(5000);
  22. We close the consumer and producer on the replyQueue
    23.            replyConsumer.close();
           replyProducer.close();
  23. We elete the temporary queue
    24.            replyQueue.delete();
  24. And finally, always remember to close your JMS connections after use, in a finally block. Closing a JMS connection will automatically close all of its sessions, consumers, producer and browser objects
    25.
