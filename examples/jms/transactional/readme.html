JMS Transactional Session Example


This example shows you how to use a transactional Session with JBoss Messaging.

Messages can be sent and received over transactional sessions, please consult the JMS 1.1 specification for full details.


Example step-by-step

To run the example, simply type ant from this directory


  1. First we need to get an initial context so we can look-up the JMS connection factory and destination objects from JNDI. This initial context will get it's properties from the client-jndi.properties file in the directory ../common/config
    2.            InitialContext initialContext = getContext();
  2. We look-up the JMS queue object from JNDI
    3.            Queue queue = (Queue) initialContext.lookup("/queue/exampleQueue");
  3. We look-up the JMS connection factory object from JNDI
    4.            ConnectionFactory cf = (ConnectionFactory) initialContext.lookup("/ConnectionFactory");
  4. We create a JMS connection
    5.            connection = cf.createConnection();
  5. We start the connection. In order for delivery to occur on any consumers or subscribers on a connection, the connection must be started
    6.            connection.start();
  6. We create a JMS session. The session is created as transacted.
    7.            Session session = connection.createSession(true, Session.SESSION_TRANSACTED);
  7. We create a JMS message producer on the session. This will be used to send the messages.
    8. 	   MessageProducer messageProducer = session.createProducer(queue);
  8. We create a JMS text message to be sent
    9.            TextMessage message = session.createTextMessage("This is a text message");
  9. We send message to the queue
    10.            messageProducer.send(message);
  10. We commit the session to let the message be delivered. After commit, the session will start a new transaction automatically.
    11.            session.commit();
  11. We create a message consumer on the queue
    12.            TextMessage message2 = session.createTextMessage("This is a text message 2");
	   
    
           MessageConsumer messageConsumer = session.createConsumer(queue);
  12. We receive the message
    13.            message = (TextMessage) messageConsumer.receive(5000);
  13. We now commit the transaction. It will finish the message delivery.
    14.  
           session.commit();
  14. And finally (no pun intended), always remember to close your JMS connections after use, in a finally block. Closing a JMS connection will automatically close all of its session, consumer, producer and browser objects
    15.            

      finally
      {
         if (connection != null)
         {
            // Be sure to close our JMS resources!
            connection.close();
         }
      }
